kwecryptfs

kwecryptfs ist ein Fontend für das ecryptfs Dateisystem, welches Verzeichnisse Dateiweise verschlüsselt.

Notiz: Benötigt root Rechte.

1. Konfiguration

Hier kann man einige Standardwerte für das Erstellen von verschlüsselten Laufwerken konfigurieren. Man kann die Vorgabewerte beim erstellen anpassen.

Attribute: Sollen die Metadaten der verschlüsselten Dateien in den erweiterten Attributen des Dateisystems gespeichert werden (ja/nein)? Dateisysteme wie ext2/3/4 unterstützen dies zum Beispiel.

Chiffrierungstyp: Wählen Sie einen Chiffrierungstyp aus. kwecryptfs unterstützt (aes, blowfish, cast5, cast6, des3_ede, twofish), falls der Kernel diese unterstützt. Beim Schlüsseltyp "passphrase" wird nur "aes", bei "tspi"e; werden alle Chiffrierungstypen unterstützt. Standard ist "aes". Der Kernel benötigt mindestens die Option CONFIG_ECRYPT_FS=m. Siehe /usr/src/linux-VERS/Documentation/filesystems/ecryptfs.txt für weitere Details.

Dateinamenverschlüsselung: Sollen die Namen der Dateien auch verschl&UUML;sselt werden (Ja/Nein)?

Mountverz: Mountverzeichnis auswählen. Dieses Verzeichnis muss schon vorher existieren und sollte in der Datei /etc/fstab konfiguriert sein. Dieses dient zum einbinden von einem externen Laufwerk, das die (verschlüsselten) Passphrase Dateien enthält.

Signaturen: Soll für das Einhängen des verschlüsselten Laufwerks Signaturen benutzt werden (Ja/Nein)? Diese heissen ecryptfs_sig und ecryptfs_fnek_sig, siehe man 7 ecryptfs.

Schlüsselgrösse: Wählen Sie bitte die Grösse, in Byte, des Schlüssels aus. Standard ist 16.

Schlüsseltyp: Wählen Sie den Schlüsseltyp (passphrase, tspi) aus. Standard ist "passphrase" und wird zur Zeit auch nur als einziger unterstützt.

Noauto: Soll das verschlüsselte Verzeichnis beim Start des Rechners manuell erstellt werden (ja/nein)? Also nicht automatisch.

Passphrase löschen: Soll beim aushängen des laufwerks auch die Passphrase aus dem Kernel Schlüsselring gelöscht werden (Ja/Nein)?

unverschlüsselte_Dateien: Sollen nicht verschlüsselte Dateien ins verschlüsselte Verzeichnis mit durchgereicht werden (ja/nein)?

Versionsanzeige: Soll die Version der verschlüsselten Dateien angezeigt werden (Ja/Nein)? Ist gut für Backup Tools die erweiterte Attribute des Dateisystems unterstützen, wie ext2/3/4.



2. Verzeichnis

Verschlüsselte Verzeichnisse erstellen, ändern oder löschen.

anlegen: Anlegen eines verschlüsselten Verzeichnis. Haben Sie in der Konfiguration die Verwendung von Signaturen aktiviert, wird die Passphrase nicht in der datei /etc/fstab gespeichert. Ansonsten könnte da zum Beispiel diese Option drin stehen:
key=passphrase:passphrase_passwd=PASSPHRASE.

ändern: Ändern von einigen Optionen für das Einhängen eines verschlüsselten Verzeichnis

löschen: Löschen von ein oder mehreren verschlüsselten Laufwerken


3. Passphrasen Verwaltung

Hier kann man Passphrase Dateien erstellen, vorhandene löschen, das Verschlüsselungspasswort einer ecryptfs Passphrase ändern oder einfach nur eine Passphrase erstellen/generieren.

anlegen:

Schlüssel Module Optionen: Auswahl einer Schlüssel Module Option. Zur Auswahl stehen passphrasse_passwd, passphrasse_passwd_file und verschlüsslte Pasphrase. Je nachdem welche Option Sie ausw&aul;hlen werden die unteren Menüpunkte angezeigt.

Dateiname: Auswahl/Eingabe eines Dateinamens für die (verschlüsselte) Passphrase.

verschlüsselte Passphrase: Eingabe einer Passphrase die verschlüsselt werden soll. Diese muss mindestens eine Zeichenlänge von 16 Zeichen haben.

Passphraseneingabe: Eingabe einer Passphrase oder eines Passworts, um zum Beispiel eine Pasphrase zu verschlüsseln. Die Zeichenlänge der Passphrase oder des Passworts muss mindestens 8 Zeichen lang sein.

Passphrase generieren: Eine Passphrase per Funktion "gen_keys" generieren.

ändern: Ändern des Verschlüsselungspassworts einer ecryptfs Passphrase Datei.

löschen: Löschen von ein oder mehreren Passphrase Dateien.
