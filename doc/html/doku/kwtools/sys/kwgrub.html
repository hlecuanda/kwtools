kwgrub

Installiert und konfiguriert den Bootloader Grub. Mit Grub brauchen Sie nicht mehr den Bootloader neu zu installieren, falls Sie einen neuen Kernel kreiert haben. Genaueres gibt es mit dem Befehl "info grub". Man kann ihn auf Diskette, in den Master Boot Record oder in den Bootsector einer Partition installieren.

Notiz: Benötigt root Rechte. Dieses Script funktioniert zur Zeit nur mit grub. grub2 wurde noch nicht getestet.

1 Boot-Konfiguration

Hier kann man entweder eine neue Konfiguration anlegen oder eine vorhandene Konfiguration auswählen und verändern oder löschen.

Name: Name der Konfiguration (neu eingeben,vorhandene auswählen).

Kernel_Part: Hier kann man eine Partition im Grub spezifischen Format auswählen, auf welcher der Kernel und die *stage* Dateien liegen.

Kernel: Auswahl des Kernels der gebootet werden soll. Wird memtest86.bin ausgewählt, benötigt man die unteren Einstellungen nicht. Es kommt dann eine Info-Meldung.

Root_Part: Hier kann man eine Partition auswählen, welche die Root-Partition ist, für diese Konfiguration.

Append: Eingabe zusätzlicher Kernel Parameter, wie vga=791 video=vesa:mtrr (alles eine Zeile)

Initrd: Auswahl der Initrd (Ramdisk) falls ihr ausgewählter Kernel eine benötigt. Nicht zu verwechseln mit Module1 (Multibootkernel).

Lock: Sollen Benutzer nur mit einem Passwort den Menüeintrag booten dürfen (ja/nein)?

Module: Auswahl eines Multiboot-Kernel. Wird zum Beispiel bei der Virtualisierungs-Software Xen benötigt.

Module1: Auswahl der initrd für den Multiboot-Kernel.

Memtest: Auswahl des Testprogramms für den Arbeitsspeicher.

Makeactive: Wird nur benötigt wenn grub einen anderen Bootloader, über das chainloading Verfahren, starten soll/muss. Zum Beispiel bei Windows. Wenn Sie ja/yes drücken wird chainloader auf +1 gesetzt. Das ist der 1. Sector auf der Boot-Partition.


2 Konfiguration

Menue anzeigen: Soll das Bootmenü beim booten angezeigt werden (ja/nein)?

Password: Password welches beim booten benötigt wird um die Bootparameter zu ändern.

Splash: "splashimage" ist eigentlich kein eigener grub Befehl. Man kann hier aber ein Splashimage auswählen, welches dann als Hintergrundbild für das Bootmenü dient. Es muss ein Bild im XPM Format sein. Näheres dazu auf der Seite:
http://ruslug.rutgers.edu/~mcgrof/grub-images/

Standard-Boot: Welcher Kernel/Eintrag soll standardmässig gebootet werden? 0 ist der erste Eintrag. Es werden alle Einträge der Reihe nach abgearbeitet.

Tastatur: Tastaturbelegung. Zur Zeit ist nur deutsch und englisch vorhanden.

Timeout: Nach welcher Zeit soll der voreingestellte Kernel gebootet werden. Siehe Standard-Boot. Geben Sie die Zeit in Sekunden an (10 = 10 sec.).

Beispiel: menu.lst


3 Installation

Kernel-Part: Auswahl der Partition auf der sich der Kernel, stage1/2 und/oder menu.lst befinden.

MBR_Floppy_Part: Wohin soll grub installiert werden? In den Master Boot Records (MBR), auf Diskette (Floppy) oder in den Bootsector einer Festplattenpartition.
